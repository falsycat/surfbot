# surfbot ユーザーストーリー

## Kanboardカラム構成

| カラム | 役割 |
|---|---|
| **Inbox** | surfbotが重要と判断したアイテムを追加する |
| **Keep** | ユーザーが後で読むために保持するアイテム |
| **Positive** | ユーザーが有用・興味深いと判断したアイテム |
| **Negative** | ユーザーが不要と判断したアイテム |

---

## ユーザーストーリー

### US-0: 設定を作成する

ユーザーは、まず最初に 以下の設定ファイルを直接編集し、情報源や興味関心などを設定する。
surfbotは、ここで設定した内容を参考に、Inboxにアイテムを追加する。

| ファイル名 | 設定内容 |
|---|---|
| `config.yaml` | kanboardへの接続情報、情報取得サイクルの間隔などの設定 |
| `feeds.yaml` | 情報源とその情報源に対する特別な指示<br>(例)この情報源のアイテムは、タイトルは原文を使うように |
| `format.md` | アイテムの作成に関する指示<br>(例)こういうタイプのアイテムは、このタグをつけて<br>(例)セキュリティ系の記事は赤色にして |
| `preferences.md` | アイテムの重要度評価に対する指示<br>(例)芸能関係に興味ないので、含めないで |

### US-1: 毎日のInboxチェック

surfbotが定時実行された後、ユーザーがKanboardのInboxを開く。
ユーザーにとって有用な情報のみが並んでいる。
各カードにはタイトル・要約・URL・タグが記載されている。

### US-2: 何もしない（放置）

ユーザーがInboxのカードを操作せず放置する。
surfbotは、次回の情報取得サイクルにおいて、放置アイテムと新着アイテムとを合わせて重要度を再評価する。
重要度が低いと判断されたアイテムは、surfbotによってそのままCLOSEされる。

### US-3: アイテムを読む

ユーザーはInboxのカードを確認し、興味がある場合、**Keep** 列に移動する。
Inboxのアイテムはsurfbotにより自動でCLOSEされてしまう可能性があるため、Keepに移動させることで、自動CLOSEを防ぐ。

興味がない場合、カードを **Negative** 列に移動する。
Negative列に移すことで、surfbotにネガティブフィードバックを要求する。
ユーザーは任意で、カードにコメントを追加することで移動の理由を伝えられる。surfbotはその理由をフィードバック学習に活用する。

### US-4: アイテムを読み終える

ユーザーはKeepのアイテムを開いて内容を読む。

有用だった場合、カードを **Positive** 列に移動する。
Positive列に移すことで、surfbotは、類似するアイテムの重要度を高く評価するようになる。(ポジティブフィードバック)
ユーザーは任意で、カードにコメントを追加することで移動の理由を伝えられる。surfbotはその理由をフィードバック学習に活用する。

無用だった場合、カードを **Negative** 列に移動する。
Negative列に移すことで、surfbotは、類似するアイテムの重要度を低く評価するようになる。(ネガティブフィードバック)

## surfbotによる自動操作

surfbotはkanboard上において、特定の列のOPENなアイテムに対してのみ、下表に示した操作を行う。
surfbotは、下表で言及されている操作以外は一切行わない。

| 列 | 操作 |
|---|---|
| Inbox | 他Inboxアイテムと比べて重要度が劣後した場合に、その旨をコメントした上で、CLOSEする |
| Keep | (surfbotの操作対象外) |
| Positive | positiveフィードバック学習後、その旨をコメントした上で、CLOSEする |
| Negative | negativeフィードバック学習後、その旨をコメントした上で、CLOSEする |

### 重要度劣後の判断基準

Inboxのアイテムが最大件数を超える場合は、全アイテムを重要度順にソートし、重要度の低いものを削除する。
最大件数は、ユーザーが`config.yaml`で設定する。

### 重複アイテムの扱い

同一URLのアイテムがKanboard上（列・状態を問わず）に既に存在する場合、surfbotはそのアイテムの追加をスキップする。

## カードのフォーマット

カードの内容は、`feeds.yaml`に記載された情報源ごとの指示と、`format.md`に記載された指示を元に決定される。

以下のフィールドは常に自動設定される：

```
external link: [元記事のURL]
start date   : [元記事の公開日時]
```

上記以外のフィールドは、`format.md` および `feeds.yaml` の自然言語指示に基づき、LLM が Kanboard のサポートする任意のフィールドを自由に設定できる。
(例) title、description、tags、color、due date など
